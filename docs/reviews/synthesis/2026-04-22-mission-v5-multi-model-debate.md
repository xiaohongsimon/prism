## 强共识（≥ 6 模型一致）

### ① NN6 "敬畏上线" 是空炮，必须补可执行的落地锚点

**7/7 全票**。

这是 v5 辩论的核心焦点。措辞分布：
- opus47: "没有可观测的失败信号，等于没有牙齿"
- grok42: "纸面纪律，几乎必然退化为'心情好就放行'"
- gemini31: "道德口号，缺乏防线闭环"
- mimo: "仅声明纪律，未定义技术实现约束"
- kimi26: "作者既是开发者又是用户，疲劳时会被瞬间绕过"
- minimax: "执行层是空的，第一次性能压力下被绕过"
- glm51: "LLM 慢了 3 倍时作者一定会注释掉门禁"

**具体落地方案建议（分布）**：

| 来源 | 方案 |
|---|---|
| opus47 | ① 一键"这条是污染"反馈入口 ② 连续失败即降级为 staging 的阈值，两者作为新 pipeline 上线前置条件 |
| grok42 | 双 LLM judge + 随机 10% 作者确认 + bypass 记录不可删 audit log + 30 天强制回顾 |
| gemini31 | 作者连续纠错 > 5 条 → 降级该 pipeline 为纯原文透传，直到修复并通过基准测试 |
| mimo | 门禁实现为 tech-stack 里**不可跳过的异步检查**，失败时写本地日志 + 暂停对应 UI 路径渲染 |
| kimi26 | 任何绕过必须写入 `../reviews/adhoc/YYYY-MM-DD-nn6-override.md`（产物、原因、回滚计划）；连续 2 次未记录绕过触发 mission 重评 |
| minimax | mission 层加"门禁形式与阈值由 tech-stack.md 明确，不接受静默降级"；tech-stack 定义门槛和绕过代价 |
| glm51 | **架构约束**：自动化产物必须先写 shadow DB 或 `draft` 态，经人工确认或独立验证后才可写主 UI 路径 |

**方案聚类**：
- **架构派**（glm51/mimo）：通过 shadow DB / 异步检查把门禁变成架构约束，绕过成本 = 改架构
- **反馈派**（opus47/gemini31）：给作者一键标记污染的入口 + 连续失败自动降级
- **审计派**（grok42/kimi26）：bypass 记录 + 周期性审查，把绕过变成有成本的显式动作
- **分层派**（minimax）：mission 只加"不接受静默降级"，细节归 tech-stack

三派兼容，最强方案 = 三派合流（架构 + 反馈 + 审计）。

### ② §5 "系统化选入机制暂缺" 必须加时间锚点

**6/7 同意加时间锚**（gemini31 是唯一反对者，理由见下）。

措辞分布：
- opus47: 绑定 `/feed` 真正开始承担推荐引擎训练数据入口时
- grok42: Wave 2 前；否则触发 mission 回滚至需重辩状态
- mimo: Wave 2 结束前（2026-Q4）交付最小可行自动选入原型
- kimi26: 绑定 roadmap Wave 2 终点
- minimax: Wave 2 实现，Wave 1 清理不被污染
- glm51: Wave 2 前必须确立半自动选入机制
- **gemini31（反共识）**：人肉选入在 NN2 算力约束 + NN6 质量优先的前提下**是信噪比最高的方案**，强行自动化违背第一性原理

**我的裁决倾向**：gemini31 的反共识有力。但"诚实写未解问题 vs 永久免责"的张力确实存在。折中：**不设硬时间锁，但加一条被动 trip-wire**——"连续 N 月（如 6 月）无新增优质源 → 重评选入机制优先级"。这同时满足 gemini31（不强制自动化）和多数派（不永久免罪）。

### ③ Trip-wires 需要为 NN3 / NN6 配套补条

**6/7 同意补充**（kimi26 是唯一反对者）。

补条建议汇总：

| 类型 | 来源 | 触发 | 行动 |
|---|---|---|---|

**我的裁决倾向**：kimi26 担心合理但不足以推翻多数。NN3/NN6 是 v5 新引入的硬约束，没有触发机制 = 约束失效。**至少加 2 条：NN6 事后标记触发器 + NN3 外泄熔断**，这是最小集。

### ④ "删除多用户 / SaaS Non-goal" 分歧最大

**4/7 认为删错了**（grok42/mimo/kimi26/glm51），**2/7 认为删对了**（opus47/gemini31），1/7 未明确表态（minimax）。

反对删的理由高度一致：
- **开源天然吸引多用户部署者**（kimi26）
- **国内典型漂移路径**：自用 → 朋友圈炫耀 → 被求分享 → 偷偷加多租户 → 系统腐化（glm51）
- **朋友试用产生需求漂移**（mimo）
- **公开性 non-goal 应当作为摩擦力保留**（grok42）

opus47/gemini31 支持删：NN1（个人工作台）+ NN3（偏好不对外）已结构性锁死，再写是冗余防御，反而稀释 non-goals 信号强度。

**我的裁决倾向**：**加回一条弱化版**——"多用户 / 多租户部署：代码可开源，但架构、issue 响应、文档都不以多用户为前提；不为权限隔离或资源争抢做设计"。理由：这是防御性摩擦条款，不是功能禁令，成本极低但对"朋友求分享"漂移路径的防守价值很高。

### ⑤ 推荐引擎（§8）需要预先声明与 NN3 / NN6 的优先级

**5/7 主动提到**（opus47/gemini31/mimo/kimi26/minimax）。

具体建议：
- opus47: 冲突优先级写死 **NN3 > NN6 > 推荐效果**，不得因效果稀释前两者
- gemini31: 在推荐引擎条目补"（严格受制于 NN2 算力上限与 NN3 隐私边界）"，锁死纯本地属性
- mimo: 训练数据仅本地聚合匿名化特征（主题偏好向量），不存原始内容
- kimi26: **NN3 扩展到模型资产**——含反推偏好的 prompt 模板、embedding 字典、特征模板保密级别等同原始偏好
- minimax: §8 措辞与 NN3 对齐——被动信号 vs 主动偏好表达分开，避免语义张力

**共识**：推荐引擎重建是 §8 未来项，但现在就要把约束边界写进 mission，否则未来做的时候会为"效果"稀释约束。

## 合并建议（作者决策清单）

### 必改（≥ 6 模型共识）

1. **NN6 补落地锚点**：mission 层加一句"门禁形式、绕过代价、失败处置由 tech-stack.md 明确，不接受静默降级"；**tech-stack.md 同步加**：shadow DB/draft 态 + 作者一键标记污染的反馈入口 + bypass 必须写 override 日志 + 连续失败自动降级为 staging

3. **§9 新增 TW4 NN3 外泄熔断**：偏好数据 / 画像 / 权重文件出现在 git diff / 日志 / 错误上报 → 立即停机 + 清理历史 + 审计
4. **§5 加被动重评触发器**（而非硬时间锁）：连续 N 月无新增优质源 → 重评选入机制优先级
5. **§8 预声明冲突优先级**：推荐引擎重建时受 NN3 / NN6 / NN2 约束，三者不得因"推荐效果"被稀释；冲突优先级 **NN3 > NN6 > 推荐效果**

### 应改（3–5 模型一致）

7. **NN3 扩展到模型资产**：含反推偏好的 prompt 模板、embedding 字典、特征向量字典保密级别等同偏好数据

## 分化点（作者决策）

### D1. NN6 落地方案选择

- **A**: 架构派（shadow DB / draft 态）+ 反馈派（一键标记）+ 审计派（bypass 日志）三派合流
- **B**: 只选最轻的——mission 加"不接受静默降级" + tech-stack 定义门槛 + bypass 日志
- **C**: 全上——加上双 LLM judge + 10% 强制人审

我倾向 **A**——落地最可靠，实现成本中等，三派机制互相兜底。C 方案在单人项目里成本过高（10% 强制人审会很快磨光作者耐心）。

### D2. §5 暂缺是"时间锁"还是"被动触发"

- **A**: 硬时间锁（Wave 2 前必须实现）
- **B**: 被动触发（N 月无新增优质源 → 重评）
- **C**: 不加任何锁（gemini31 立场：人肉就是最优解）

我倾向 **B**——同时回应 gemini31 的"不强制自动化"和多数派的"不永久免罪"。

### D3. 多用户 non-goal 回不回

- **A**: 加回弱化版"代码可开源但架构不做多用户"
- **B**: 维持 v5 删除状态
- **C**: 加回硬禁令

我倾向 **A**——防御性摩擦条款成本极低，但 4/7 模型警示的漂移路径真实存在。

### D4. 是否预先声明 NN3/NN6/推荐引擎优先级

- **A**: 在 §8 里显式写 NN3 > NN6 > 推荐效果
- **B**: 不预先声明，等真做时再辩论

我倾向 **A**——预先声明成本低，未来做推荐引擎时稀释约束的冲动真实存在。

## 最隐蔽的盲点（多模型不约而同）

**"作者即唯一评审 + 第二大脑共生 + 信息茧房固化 + 注意力零边际幻觉"——四个不同角度指向同一个盲点：v5 的所有机制都预设"作者是清醒、警觉、有选择权的评审者"，但随着 Prism 运行时间变长，作者会逐渐失去这种评审能力**。

opus47 叫它"认知同化"，grok42 叫"世界观被缓慢重塑"，gemini31 叫"回音室固化"，glm51 叫"虚假饱腹感"——都是同一件事。

**v5 的 mission 目前对此没有任何对抗机制**。NN6 防的是"错误产物污染"，但防不了"大量正确但同质产物的缓慢塑形"。

这是 v5 之后 mission 层最值得加的一条——可能叫 **NN7 保持外部性** 或 **TW5 反习惯化强制复核**：

> 月度强制抽样 N 条已消费内容回原文核对；季度强制跳出 Prism 使用外部未被个性化的信息源（如订阅纸质报刊、听非英文圈播客）若干小时。

但这已经超出 mission 的技术契约范围，进入"作者使用守则"领域——是否写进宪章值得再议。
